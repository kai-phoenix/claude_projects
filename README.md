# Learning OS Repository

このリポジトリは **AI活用による学習OS** です。

## 📌 このリポジトリについて

- `/learning` 以下に、学習目的のミニプロジェクトを無限生成・管理するために存在します
- 言語・フレームワークを問わず、「小さな本番風構成を高速回転させる」ことが目的です
- リポジトリ直下の `CLAUDE.md`（憲法）と `/learning/CLAUDE.md`（実務ルール）の2層構成で統一運用されています

**このリポジトリ自体に開発対象のコードはありません。** 全ての開発は `/learning` 配下のプロジェクトで行われます。

## 📚 ドキュメント構成

| ファイル | 役割 |
|---------|------|
| **CLAUDE.md** | リポジトリ全体の憲法（親ルール）：レビュー・設計・セキュリティ・自動生成制限・Git操作方針 |
| **learning/CLAUDE.md** | 学習プロジェクト生成の実務ルール：新規プロジェクト立ち上げフロー・技術別ルール・新技術モード |

## 🚀 使い方（クイックガイド）

### 新しい学習プロジェクトを作りたい

1. ユーザーが「◯◯の学習プロジェクト作って」と依頼
2. ClaudeCode は `learning/CLAUDE.md` に従い、**構成案を提示**
3. ユーザーが「OK」と承認
4. ClaudeCode が **最小限のファイル**を生成
5. ユーザーが実装・学習
6. 学習内容を `learning/CLAUDE.md` に追記（自己強化）

## 🎯 設計哲学

- **完成度より回転率** - 高い品質より高速サイクルを優先
- **骨組みはAI、実装は人間** - AIは提案と補助に徹し、実装はユーザーが担当
- **学習ログを資産化** - 各プロジェクトの学習成果をGitで管理し、次の学習に活かす

## 📋 プロジェクト完成の定義

学習用プロジェクトが「完成」とみなされる条件:

- README.md に目的・実行方法が明記されている
- 最初のテストが1つ以上実行でき、パスしている
- src/ 以下に最小限の実装が存在する（ダミーコードは不可）
- セキュリティレビュー対象項目をチェック済み
- 依存ライブラリのバージョンが明記されている
- Git コミットメッセージが日本語で意図が明確
- `.env.example` が存在する（秘密情報が必要な場合）

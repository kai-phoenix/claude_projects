# /learning用CLAUDE.md
このファイルは「/learning配下で作られる全ての学習プロジェクト」の生成方針を定義する。
親の`/.Claude.md`の方針を継承しつつ、この階層に特化したルールをここに書く。

# 1.　このディレクトリの目的
- 学習用のミニプロジェクトを無限に積み上げる場所
- 日付+技術+テーマの形式で管理する
- 完成度より回転率を重視し、小さな本番風構成の学習量産を目的とする

例:
learning/
├─ 20251128_laravel_service_layer/
├─ 20251130_nextjs_auth_demo/
└─ 20251201_python_batch/

# 2. 学習プロジェクト生成ルール
私が:
> 「Laravel の学習用プロジェクトを作って」  
> 「Next.js の練習テーマほしい」  
> 「Python CLI の課題作って」  

などと言った時、ClaudeCode は次の手順に従う。
## 2-1. ディレクトリ名の提案
形式：
`YYYYMMDD_<tech>_<theme>`

例：
- `20251201_laravel_api_search`
- `20251202_nextjs_form_validation`
- `20251203_python_log_parser`

## 2-2. 最小構成の構成案を提示(実行前に必須)
各プロジェクトには最低限次を含めること:
- `README.md`
  - 目的
  - 学習テーマ
  - 使用技術
  - 今日やること(TODO)
  -  テスト実行方法 
- `src/`
  - 技術に応じた最小骨組み
- `tests/`
  - その技術にあったテストの雛形(最低一つ)

※ テンプレ不要。Claudeが都度判断して最適な骨組みを作る。

## 2-3. OKが出たら生成開始
- 構成案→OK確認→生成を厳守する

## 2-4. 技術別基準 (簡易ルール)
テンプレを持たない代わりに、ここで「軽量な基準」を定義する。

### Laravelの場合
- Controller/Service/Repositoryの３層構造推奨
- Requestクラスでバリデーション
- Feature+Unitテストを一つづつ

### pythonの場合
- CLI or スクリプトは`src/main.py`
- pytestを使用してテスト一つ作成

### Next.jsの場合
- app/構成+api route
- components/に責務分離
- React Testing Libraryのテストひとつ

### Dockerを含む場合
- docker.compose.yml(app+dbの最小構成)
- 環境変数は `.env.example` を生成  
- 不要なコンテナは増やさない

### AWS 要素が必要な場合
- 可能なら構成図を README に文章で添える  
- CloudFormation/CDK は“最小サンプル”だけでOK  
- セキュリティグループは必要最小限に限定する

ClaudeCodeはこの基準を踏まえて骨組みを作る。

## 2-5. 新技術モード(最重要)
未知の技術（Rust, Go, Deno, Bun, SolidJS, Astro, Zig, Kotlin…）を学習するときのルール。
ClaudeCode は、テンプレが存在しない技術を次の方針で扱うこと。

### 完全新技術プロジェクト生成ルール
初見技術のときは、次の「標準フロー」でプロジェクト案を作る：

1. **Hello World + α** の難易度に固定  
   - CLIツール  
   - 単一API  
   - 小さなWebサーバ  
   のいずれかに絞る  
2. その技術の **公式が推奨する標準構成** を最優先で使う  
   - Rust → Cargo構成  
   - Go → 標準go build構成  
   - JS → Node or Bun標準  
3. テストは **その技術の標準手法** を使い最低1つ入れる  
4. README に必ず次を含める：  
   - セットアップ  
   - 実行コマンド  
   - テストコマンド  
5. 設計は凝らず、  
   **「その技術での基本API・文法・型・エラーハンドリング」に慣れる**ことを優先  

### 初回学習後の流れ
初回プロジェクト終了後、ClaudeCode は私が望めば：
- この /learning/CLAUDE.md に  
  **その技術の“軽い方針”だけを追記するためのドラフト** を生成する  
（例：Rust向け軽量ルール、Go向け軽量ルールなど）
→ この“自己強化ループ”により、学習回数が増えるほどこのファイルが進化し、  
新技術の導入がどんどん楽になる。

### 運用フローまとめ
1. **新技術触りたい！**  
2. ClaudeCodeに：「◯◯完全初見。学習用プロジェクト作って」  
3. ClaudeCodeが：  
   - ディレクトリ名案  
   - 最小構成案  
   - READMEドラフト  
   を提示  
4. 私：「OK」  
5. ClaudeCodeがファイル生成  
6. コーディング → レビュー → 学習  
7. Claudeに：「今回学んだ内容を /learning/CLAUDE.md に追加するルール案ちょうだい」  
8. ファイルの方針を育てていく（自己強化）

# 3. 親ルール
レビュー・設計・セキュリティ・テストの横断ポリシーは  
リポジトリ直下の `.CLAUDE.md` を常に最優先で参照する。

# 4. 言語設定
この階層での全ての対話と指示は日本語で行う。
ClaudeCodeは確認・合意のメッセージも日本語で返すこと。
